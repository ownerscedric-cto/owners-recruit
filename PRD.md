# 오너스경영연구소 입사 지원자 정보 관리 시스템 PRD
*Product Requirements Document*

---

## 1. 제품 개요

### 1.1 제품명
오너스경영연구소 스마트 리크루팅 시스템 (OWNERS Smart Recruiting System)

### 1.2 비전
"효율적이고 안전한 입사 프로세스를 통해 인재와 기업을 연결하는 디지털 게이트웨이"

### 1.3 미션
- 입사 예정자의 정보 제출 프로세스를 디지털화하여 편의성 향상
- 인사팀의 데이터 처리 업무를 자동화하여 업무 효율성 극대화
- 개인정보 보안 강화를 통한 신뢰성 있는 입사 시스템 구축

### 1.4 제품 포지셔닝
보험설계사 전문 인력을 관리하는 오너스경영연구소만의 특화된 입사 관리 시스템으로, 업계 특성을 반영한 맞춤형 솔루션

---

## 2. 문제 정의

### 2.1 현재 상황 (As-Is)
#### 문제점 1: 비효율적인 정보 수집 프로세스
- 입사 예정자들이 개인정보를 카카오톡 메시지로 개별 전송
- 인사팀 직원이 수동으로 복사/붙여넣기 작업 수행
- 정보 누락 및 오류 발생 가능성 높음
- 표준화되지 않은 데이터 형식으로 인한 처리 어려움

#### 문제점 2: 보안 취약성
- 카카오톡을 통한 민감 개인정보 전송으로 보안 위험 노출
- 정보 유출 시 추적 및 관리 불가능
- GDPR 및 개인정보보호법 준수 어려움

#### 문제점 3: 업무 프로세스 비효율성
- 인사팀 → 선임자 → 인사팀 → 총무팀의 복잡한 정보 전달 체계
- 중복 작업 및 시간 지연 발생
- 실시간 진행 상황 파악 불가능

### 2.2 목표 상황 (To-Be)
- 표준화된 웹 폼을 통한 체계적인 정보 수집
- 자동화된 데이터 처리 및 관리
- 보안이 강화된 정보 전송 및 저장
- 실시간 모니터링 및 효율적인 업무 프로세스

## 3. 타겟 사용자

### 3.1 Primary Users

#### 페르소나 1: 입사 예정자 (보험설계사)
- **연령대**: 30-50대
- **기술 수준**: 중급 (스마트폰 및 기본 웹 사용 가능)
- **니즈**:
  - 간편한 정보 입력 방식
  - 명확한 입사 절차 안내
  - 필요 서류 및 교육 정보 제공
- **Pain Points**:
  - 복잡한 입사 절차에 대한 불안감
  - 필요 서류 준비 과정의 어려움
  - 진행 상황 파악 불가

#### 페르소나 2: 인사팀 직원
- **역할**: 입사 프로세스 관리 및 데이터 처리
- **니즈**:
  - 효율적인 데이터 수집 및 관리
  - 자동화된 보고서 생성
  - 총무팀과의 원활한 협업
- **Pain Points**:
  - 반복적인 수동 작업
  - 데이터 정확성 검증 어려움
  - 진행 상황 추적 관리 부담

### 3.2 Secondary Users

#### 페르소나 3: 모집인 (선임자)
- **역할**: 입사 예정자 모집 및 초기 안내
- **니즈**:
  - 모집한 입사 예정자 진행 상황 확인
  - 간편한 정보 전달 방법

#### 페르소나 4: 총무팀 직원
- **역할**: 최종 입사 처리 및 행정 업무
- **니즈**:
  - 정확하고 완전한 입사자 정보 수신
  - 표준화된 데이터 형식

## 4. 핵심 가치 제안 (Value Proposition)

### 4.1 입사 예정자를 위한 가치
- **편의성**: 언제 어디서나 접속 가능한 웹 기반 신청
- **투명성**: 입사 프로세스 전체 과정 가시화
- **지원**: 필요 교육 및 서류 준비 가이드 제공
- **안심**: 개인정보 보안 강화

### 4.2 인사팀을 위한 가치
- **효율성**: 업무 시간 80% 단축
- **정확성**: 데이터 오류율 95% 감소
- **자동화**: 반복 작업 자동 처리
- **통제**: 실시간 모니터링 및 관리

### 4.3 조직 전체를 위한 가치
- **컴플라이언스**: 개인정보보호 규정 준수
- **생산성**: 전체 입사 프로세스 기간 50% 단축
- **데이터 기반 의사결정**: 입사 관련 통계 및 인사이트 제공

## 5. 주요 기능

### 5.1 입사 예정자용 기능

#### 5.1.1 정보 입력 시스템
- **기본 정보 입력 폼**
  - 개인정보 (이름, 연락처, 주소 등)
  - 경력 정보
  - 자격증 정보
  - 서류 업로드 기능
- **단계별 입력 가이드**
  - 진행 상황 표시 (Progress Bar)
  - 필수/선택 항목 구분
  - 실시간 유효성 검증

#### 5.1.2 입사 가이드 시스템
- **입사 프로세스 안내**
  - 전체 입사 절차 시각화
  - 단계별 상세 설명
  - 예상 소요 시간 안내
- **교육 정보 제공**
  - 필수 이수 교육 목록
  - 교육 신청 링크 연결
  - 교육 완료 확인 기능
- **서류 준비 가이드**
  - 필요 서류 체크리스트
  - 서류 발급 방법 안내
  - 경력자용 말소 확인증 발급 사이트 연결

#### 5.1.3 진행 상황 확인
- 신청 상태 실시간 조회
- 부족한 정보/서류 알림
- 예상 처리 일정 안내

### 5.2 인사팀용 기능

#### 5.2.1 대시보드
- **실시간 현황 모니터링**
  - 신규 신청자 수
  - 처리 대기 건수
  - 완료 건수
  - 주요 지표 시각화
- **알림 시스템**
  - 신규 신청 알림
  - 미처리 건 알림
  - 이상 징후 감지

#### 5.2.2 신청자 관리 시스템
- **목록 조회 및 필터링**
  - 다양한 검색 조건
  - 상태별 분류
  - 날짜별 정렬
- **상세 정보 조회**
  - 개인별 상세 페이지
  - 제출 서류 다운로드
  - 수정 이력 추적
- **일괄 처리 기능**
  - 다중 선택 처리
  - 상태 일괄 변경
  - 그룹 메시지 발송

#### 5.2.3 데이터 처리 기능
- **엑셀 내보내기**
  - 커스터마이징 가능한 템플릿
  - 총무팀 전달용 포맷
  - 자동 데이터 정합성 검증
- **보고서 생성**
  - 주간/월간 리포트
  - 모집인별 실적 통계
  - 입사 트렌드 분석

#### 5.2.4 커뮤니케이션 기능
- **자동 이메일/SMS 발송**
  - 신청 완료 확인
  - 추가 정보 요청
  - 처리 완료 안내
- **내부 협업 도구**
  - 담당자 지정
  - 코멘트 기능
  - 처리 이력 기록

### 5.3 관리자 기능

#### 5.3.1 시스템 설정
- 입력 폼 커스터마이징
- 프로세스 단계 설정
- 권한 관리

#### 5.3.2 보안 관리
- 접근 권한 설정
- 로그 모니터링
- 데이터 백업 관리

## 6. 사용자 경험 (UX)

### 6.1 디자인 원칙
- **간결성**: 필수 정보만을 요구하는 미니멀한 인터페이스
- **명확성**: 직관적인 레이블과 도움말 제공
- **반응성**: 모바일 우선 반응형 디자인
- **접근성**: WCAG 2.1 AA 준수

### 6.2 사용자 여정

#### 입사 예정자 Journey
1. **접속**: 제공받은 링크를 통해 시스템 접속
2. **안내**: 입사 프로세스 전체 개요 확인
3. **입력**: 단계별 정보 입력 및 서류 업로드
4. **확인**: 입력 정보 검토 및 제출
5. **추적**: 처리 상태 확인 및 알림 수신

#### 인사팀 Journey
1. **모니터링**: 대시보드에서 신규 신청 확인
2. **검토**: 제출된 정보 및 서류 검증
3. **처리**: 승인/반려/보완 요청 처리
4. **전달**: 총무팀으로 데이터 전송
5. **완료**: 입사 처리 완료 및 기록

### 6.3 인터페이스 가이드라인
- 일관된 색상 체계 및 타이포그래피
- 명확한 CTA(Call-to-Action) 버튼
- 적절한 피드백 메시지
- 로딩 상태 표시

## 7. 기술 요구사항

### 7.1 기능 요구사항
- **플랫폼**: 웹 기반 (크로스 브라우저 지원)
- **반응형**: 모바일, 태블릿, 데스크톱 대응
- **언어**: 한국어 지원
- **파일 업로드**: 최대 10MB, PDF/JPG/PNG 포맷

### 7.2 비기능 요구사항

#### 보안
- SSL/TLS 암호화 통신
- 개인정보 암호화 저장
- 정기적인 보안 감사
- 접근 권한 관리 (RBAC)
- 2단계 인증 옵션

#### 성능
- 페이지 로드 시간 < 3초
- 동시 접속자 500명 지원
- 99.9% 가용성
- 자동 확장 가능한 인프라

#### 호환성
- Chrome, Safari, Edge 최신 2개 버전 지원
- iOS 13+, Android 10+ 지원

### 7.3 데이터 요구사항
- 데이터 보존 기간: 5년
- 일일 자동 백업
- 재해 복구 계획 수립
- GDPR 및 한국 개인정보보호법 준수

## 8. 성공 지표 (KPIs)

### 8.1 효율성 지표
- **처리 시간 단축률**: 기존 대비 80% 감소 목표
- **자동화율**: 전체 프로세스의 70% 이상 자동화
- **오류율**: 데이터 입력 오류 95% 감소

### 8.2 사용자 만족도 지표
- **입사 예정자 만족도**: NPS 40 이상
- **인사팀 만족도**: 시스템 유용성 평가 4.5/5.0 이상
- **시스템 사용률**: 대상자의 95% 이상 사용

### 8.3 비즈니스 지표
- **입사 프로세스 완료율**: 90% 이상
- **평균 처리 기간**: 3일 → 1일 단축
- **운영 비용**: 연간 운영 비용 50% 절감

### 8.4 측정 방법
- 시스템 로그 분석 (월간)
- 사용자 설문조사 (분기별)
- A/B 테스트 진행
- 정기적인 성과 리뷰 회의

## 9. 제약사항 및 리스크

### 9.1 제약사항
#### 기술적 제약
- 레거시 시스템과의 통합 한계
- 초기 개발 예산 제한
- 기존 인프라 활용 필요

#### 조직적 제약
- 변화 관리에 대한 저항
- 교육 및 적응 기간 필요
- 단계적 도입 필요

#### 규제적 제약
- 개인정보보호법 준수 필수
- 보험업 관련 규정 준수
- 전자서명법 관련 검토 필요

### 9.2 리스크 및 완화 방안

#### 리스크 1: 사용자 채택률 저조
- **발생 가능성**: 중간
- **영향도**: 높음
- **완화 방안**:
  - 사용자 교육 프로그램 운영
  - 단계적 전환 계획 수립
  - 인센티브 제공

#### 리스크 2: 보안 사고 발생
- **발생 가능성**: 낮음
- **영향도**: 매우 높음
- **완화 방안**:
  - 정기적인 보안 감사
  - 침입 탐지 시스템 구축
  - 사고 대응 매뉴얼 수립

#### 리스크 3: 시스템 장애
- **발생 가능성**: 낮음
- **영향도**: 높음
- **완화 방안**:
  - 이중화 구성
  - 정기적인 백업
  - 장애 대응 프로세스 수립

### 9.3 의존성
- IT 인프라팀의 지원
- 법무팀의 규정 검토
- 총무팀과의 협업 체계 구축
